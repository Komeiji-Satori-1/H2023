# AD9833 锁频架构设计稿

## 1. 目标

本方案将原有“DAC 直接输出采样点波形”的实现，调整为“AD9833 参数化输出 + 三 ADC 同步反馈 + 数字频率锁定”架构。

核心目标如下：

1. 保留现有 `C` 信号波形识别流程。
2. 由识别结果驱动双通道 AD9833 输出 `A`、`B` 两路信号。
3. 通过 TIM8 同步触发三路 ADC，分别采集 `C`、`A_feedback`、`B_feedback`。
4. 锁定目标不是相位字本身，而是输出频率。
5. 闭环控制量由“相位误差”换算得到，最终作用到 AD9833 频率字。

## 2. 结论

这套系统本质上是 `FFT 识别 + 三 ADC 反馈 + PID 频率跟踪`，属于锁频环路而不是纯相位环路。

推荐控制链路如下：

```text
C 信号识别
→ 1024 点 FFT 识别 A/B 基波频率和波形类型
→ 配置 AD9833 两路输出
→ 三 ADC 同步采样
→ 锁频阶段对已知频点做单频 DFT
→ 计算相位差
→ PID 输出频率修正量
→ 重写 AD9833 频率字
→ 循环跟踪
```

- `Myfun/state.c`

- `Myfun/Waveform_classification.c`

- `FFT/FFT.c`

## 4. 建议的状态机

建议在 `Myfun/state.c` 中把原流程调整为：

```text
STATE_IDLE
STATE_CHECK_HMI
STATE_SEPARATE_START
STATE_AD9833_CONFIG
STATE_FREQ_LOCK
STATE_CALC_PHASE
```

说明：

- `STATE_SEPARATE_START`
  - 采集 `C` 信号，做长帧 FFT 识别。
- `STATE_AD9833_CONFIG`
  - 根据识别结果写入 AD9833 频率字、波形类型和初始配置。
- `STATE_FREQ_LOCK`
  - 三 ADC 同步采样，执行锁频闭环。
- `STATE_CALC_PHASE`
  - 仅用于 HMI 参数更新或初始标定，不作为主闭环状态。

## 5. 三 ADC 采样组织

建议采用三个 ADC，由同一个 TIM8 事件触发：

- `ADC1 / PA0`：采集混合参考信号 `C`
- `ADC2 / PA1`：采集 AD9833_A 反馈
- `ADC3 / PA2`：采集 AD9833_B 反馈

实现建议：

1. 三个 ADC 使用相同触发源。
2. 三个 DMA 缓冲区长度一致。
3. 以同一帧编号作为时间对齐基准。

这样在软件层可以直接按“同帧”比较相位，不需要做 A/B 之间的谱解混。

## 6. 锁频算法

### 6.1 识别阶段

对 `C` 通道做 1024 点 FFT：

1. 找出两个最大基波峰值。
2. 得到 `fA`、`fB`。
3. 判断波形类型。
4. 记录初始频率基准 `baseFreqA`、`baseFreqB`。

### 6.2 锁频阶段

每次拿到一组三 ADC 同步帧后：

1. 对 `C` 通道，在 `fA` 和 `fB` 处分别做单频 DFT。
2. 对 `A_feedback` 通道，在 `fA` 处做单频 DFT。
3. 对 `B_feedback` 通道，在 `fB` 处做单频 DFT。
4. 得到四个相位量：
   - `phi_C_A`
   - `phi_A_FB`
   - `phi_C_B`
   - `phi_B_FB`
5. 计算相位误差：
   - `errA = wrap(phi_A_FB - phi_C_A)`
   - `errB = wrap(phi_B_FB - phi_C_B)`
6. 进入 PID/FLL：
   - `deltaFreqA = PID_A(errA)`
   - `deltaFreqB = PID_B(errB)`
7. 更新 AD9833 频率字。

### 6.3 控制目标

锁频不是直接让相位误差为零一次完成，而是通过相位误差的持续变化来修正频率偏差。

因此：

- 相位误差是观测量。
- PID 输出是频率修正量。
- AD9833 的频率字是最终执行量。

建议新增独立模块，不要把闭环逻辑继续塞进 AD9833 驱动层。

- `Myfun/ad9833_fll.h`

## 8. AD9833 驱动层修改建议

当前 `AD9833/AD9833.c` 中的 `ad9833_set_freq_ch()` 会在写频率字前后执行复位流程。

对于锁频闭环来说，这种做法不适合放在高频更新路径里，因为会引入不必要的输出中断。

建议新增“在线更新”接口：

```c
void ad9833_set_freq_word_live(uint32_t freq_word, uint16_t type, uint8_t ch);
void ad9833_set_freq_ch_live(uint32_t freq_hz, uint16_t type, uint8_t ch);
```

原则：

- 初始配置阶段可以用带复位的写法。
- 锁频阶段只更新频率字，不重复复位。
- 相位寄存器只用于初始化或标定，不作为主闭环量。

## 9. FFT / 单频 DFT 使用方式

锁频阶段建议继续复用：

```c
void FFT_SingleFreqDFT_U16(const uint16_t *adc_buf,
                           uint16_t len,
                           float fs,
                           float target_freq,
                           FFT_SingleFreqResult_t *result);
```

使用策略：

- 识别阶段：长帧 FFT，找峰值。
- 锁频阶段：固定频点单频 DFT，不再重复找峰。

这样可以显著降低计算量，并避免锁频阶段重新搜索频点带来的抖动。

## 10. 频差较大时的处理

虽然本方案以锁频为目标，但仍建议保留两个工作模式：

1. `COARSE_PULLIN`
   - 初始频差较大时使用。
   - PID 输出限幅较大，允许快速拉近频率。
2. `FINE_TRACK`
   - 接近锁定后使用。
   - PID 输出限幅较小，减少抖动和频率跳变。

切换条件可由以下指标决定：

- 相位误差连续多帧同向增长。
- 相位误差超过阈值并持续若干帧。
- 频率修正量连续触顶。

## 11. 建议的代码落点

### 11.1 `Myfun/state.c`

职责：

- 处理 HMI 命令。
- 调度识别与锁频状态。
- 读取三 ADC 同步帧。
- 调用识别模块和 FLL 模块。

### 11.2 `Myfun/ad9833_fll.c`

职责：

- 保存 `A/B` 两路锁频状态。
- 计算相位误差。
- 调用 PID。
- 生成频率修正量。
- 调用 AD9833 在线更新接口。

### 11.3 `AD9833/AD9833.c`

职责：

- 只负责 DDS 寄存器写入。
- 保持芯片级驱动纯净。
- 提供初始配置和在线频率更新接口。

## 12. 风险点

1. `AD9833` 频率更新如果每次都复位，会破坏闭环连续性。
2. 三 ADC 采样若触发不同步，会直接影响相位误差的可靠性。
3. 锁频环路对 PID 参数敏感，初版需要保守限幅。
4. `C` 通道单频 DFT 的目标频点必须来自识别阶段，不能在锁频阶段重新漂移搜索。

## 13. 验收标准

1. `C` 信号识别结果稳定，可得到 `fA`、`fB` 和波形类型。
2. AD9833 两路输出可按识别结果启动。
3. 三 ADC 可在同一 TIM8 触发下同步采样。
4. 锁频阶段能持续输出相位误差和频率修正量。
5. AD9833 频率字可被连续更新，输出无明显复位式中断。
6. 系统能把输出频率拉回目标值并维持稳定。
