# 2026-07-20 AD9833 FLL implementation note

## Goal

Switch the main output architecture from DAC NCO generation to AD9833 frequency tracking:

- keep the existing C-signal waveform recognition flow
- configure two AD9833 outputs from the recognized A/B signal parameters
- reserve a three-ADC feedback frame interface for C, AD9833_A, and AD9833_B
- use phase error as the PID input and update AD9833 frequency words as the actuator

## Summary

- Added a reusable PID controller.
- Added `ad9833_fll` as the frequency-lock loop module.
- Added AD9833 GPIO initialization in `ad9833_init()`.
- Added live AD9833 frequency update APIs that do not reset the DDS during tracking.
- Reworked the state machine to use:
  - `STATE_SEPARATE_START`
  - `STATE_AD9833_CONFIG`
  - `STATE_FREQ_LOCK`
- Added weak placeholder symbols for future ADC2/ADC3 feedback buffers:
  - `ADC_AD9833_A`
  - `ADC_AD9833_B`
  - `adc_fll_ready_offset`
  - `adc_fll_ready_sample_start`
